# Dive Trainer - 潛水訓練平台

> 一個專為潛水教練與學員設計的智能訓練與教學管理平台

## 📋 專案簡介

**Dive Trainer** 是一個以「閉氣訓練」為起點，逐步發展為全方位潛水訓練與教學管理平台的 Web 應用程式。本專案旨在解決潛水教練在教學過程中面臨的效率問題，並為學員提供系統化的訓練追蹤與溝通管道。

### 核心定位

**Dive Trainer** 不僅僅是一個訓練工具，更是一個**連接教練與學員的智能教學生態系統**。我們的願景是成為**台灣乃至世界級的潛水訓練與教學平台**，讓每一位潛水教練都能更有效率地進行教學，讓每一位學員都能獲得個人化的訓練指導。

---

## 🎯 目標對象

### 教練端
- **統一管理平台**：取代分散的 LINE 記事本，集中管理所有學員的訓練紀錄
- **快速搜尋與篩選**：透過學員名稱、標籤（HashTag）、分類等方式快速找到特定學員
- **訓練菜單設計**：為每位學員開立個人化的訓練菜單
- **進度追蹤**：即時掌握每位學員的練習進度與表現
- **作業批改與建議**：提供訓練紀錄的批改功能與專業建議

### 學員端
- **訓練執行**：使用教練開立的訓練菜單進行練習
- **進度查看**：隨時查看自己的訓練紀錄與進度
- **教練溝通**：接收教練的課程總結、建議與訓練菜單
- **個人化體驗**：根據個人特質與需求獲得量身打造的訓練內容

## 🚀 專案發展歷程

### 第一階段：閉氣訓練（目前階段）

本專案從「閉氣訓練」功能著手開發，主要參考並改進 **Apnea Trainer** APP 的核心功能。選擇此方向的原因：

- **專業合作夥伴**：與閉氣專業國手「洲洲」合作，其目前使用 Apnea Trainer 進行訓練與教學
- **明確需求**：閉氣訓練是潛水訓練的基礎，有明確的訓練模式與需求
- **技術可行性**：訓練流程清晰，適合作為平台開發的起點

**目前功能**：
- ✅ O2-table（提升耐低氧能力）
- ✅ CO2-table（提升耐高二氧化碳能力）
- ✅ 自訂訓練表格（Table）編輯
- ✅ 訓練計時器與進度追蹤
- ✅ 橫膈膜抽動記錄
- ✅ 離線使用支援（PWA）

### 第二階段：教練-學員管理系統（規劃中）

- 📋 學員資料管理（基本資訊、個性特質、害怕事項、喜好項目等）
- 📝 訓練菜單開立與分配
- 📊 訓練進度追蹤與分析
- 💬 教練-學員溝通平台
- 🔍 快速搜尋與篩選功能
- 🏷️ 標籤（HashTag）系統

### 第三階段：數據分析與 AI 應用（未來）

- 📈 數據搜集與分析
  - 學員基本資料（身高、體重、血型、性別等）
  - 訓練表現數據
  - 教練建議與學員問題記錄
- 🤖 AI 輔助分析
  - 數據關聯性分析
  - 個人化訓練建議
  - 常見問題自動回覆
- 🌐 平台化發展

## 💡 專案願景與定位思考

### 當前定位
**「潛水訓練與教學平台」** - 專注於教練與學員之間的訓練管理與溝通

### 願景方向
我們希望 Dive Trainer 能夠成為：
- 🌍 **台灣及世界級的潛水訓練平台**
- 🔗 **連接所有潛水教練、潛水員的社群生態系統**
- 📚 **潛水知識與經驗的共享平台**

### 定位思考
我們意識到，如果無法用精簡的方式表達平台定位，代表我們對其定位還不夠清晰。目前我們認為平台的面向應該**「更大、更廣闊」**，但仍在思考如何更精準地定義。

**可能的定位方向**：
- 潛水訓練與教學生態系統
- 智能潛水教練助手平台
- 全方位潛水學習與訓練平台

我們歡迎任何有助於釐清平台定位的建議與討論。

## 📝 數據搜集計劃

為了建立有效的 AI 分析與個人化服務，我們計劃搜集以下數據：

### 基本資料
- 身高、體重、血型、性別
- 年齡、職業、運動背景

### 訓練相關
- 訓練表現數據
- 訓練頻率與持續時間
- 訓練目標與進度

### 個人特質（由教練填寫）
- 個性特質
- 害怕的事項
- 喜歡的訓練項目
- 學習風格

### 互動數據
- 教練建議與回饋
- 學員常見問題
- 教練面臨的挑戰

**搜集方式思考**：
我們希望以**結構化、有目的性**的方式搜集數據，而非單純地收集所有平台內容。初期將專注於基本資料與訓練表現數據的搜集，逐步擴展至更複雜的個人化資訊。
